# 宠物智能关怀系统 — 需求规格说明书

## 1. 项目概述

### 1.1 项目名称
宠物智能关怀系统（PetCare）

### 1.2 技术栈
- **开发框架**：HarmonyOS ArkTS
- **目标设备**：Phone（手机/平板）
- **配套硬件**：Hi3861 开发板（温湿度传感器 + 蜂鸣器 + LED 指示灯）
- **通信方式**：WiFi + HTTP（预留）

### 1.3 项目定位
面向宠物主人的一站式智能关怀工具，融合**宠物健康管理、行为监测、档案记录、虚拟互动、领养服务**五大核心场景，支持与硬件开发板联动实现环境实时监测与告警。

## 2. 功能需求

### 2.1 用户认证模块

#### 2.1.1 登录（LoginPage）
| 项目 | 描述 |
|------|------|
| 入口 | App 启动后首页 |
| 输入项 | 用户名、密码 |
| 功能 | 记住密码（Toggle 开关） |
| 校验规则 | 用户名和密码不能为空 |
| 登录逻辑 | 检查用户名是否已注册 → 验证密码 → 成功则进入首页 |
| 失败提示 | 统一提示「用户名或密码不正确」 |
| 成功处理 | 保存当前登录用户 → 跳转至 TabBar 主页 |
| 跳转 | 点击「立即注册」→ 跳转注册页 |
| 存储方式 | UserStore（AppStorage 内存存储） |

#### 2.1.2 注册（RegisterPage）
| 项目 | 描述 |
|------|------|
| 输入项 | 用户名、宠物昵称、密码、确认密码 |
| 校验规则 | 用户名 3-12 位；密码 ≥6 位；两次密码一致；必须同意用户协议 |
| 注册逻辑 | 检查用户名是否已存在 → 保存用户凭证 |
| 重复注册 | 提示「该用户名已被注册」 |
| 成功处理 | 提示「注册成功！请登录」→ 返回登录页 |

- 恢复正常 → 警告消失

## 3. 服务层

### 3.1 UserStore（用户存储服务）
| 方法 | 功能 |
|------|------|
| `register(username, password)` | 注册用户，检查重复 |
| `login(username, password)` | 登录验证 |
| `setCurrentUser(username)` | 保存当前登录用户 |
| `getCurrentUser()` | 获取当前登录用户名 |

- **存储方式**：AppStorage（内存存储，App 退出后数据丢失）
- **存储结构**：`user_<username>` → password；`_currentUser` → 当前用户名

### 3.2 DeviceService（设备通信服务）— 预留
| 方法 | 功能 |
|------|------|
| `fetchSensorData()` | GET /api/sensor → 获取温湿度 |
| `sendAlertCommand(on/off)` | POST /api/alert → 控制蜂鸣器+LED |

### 3.3 SensorMonitor（传感器监控服务）— 预留
| 功能 | 描述 |
|------|------|
| 定时轮询 | 每 2s 请求一次传感器数据 |
| 阈值告警 | 超标 → 发送 alertOn；恢复 → 发送 alertOff |
| 状态机 | 避免重复发送告警指令 |

## 6. 硬件联动（预留）

### 6.1 硬件架构
```
手机 App（北向） ←→ WiFi/HTTP ←→ Hi3861 开发板（南向）
                                    ├── DHT11 温湿度传感器
                                    ├── 蜂鸣器
                                    └── LED 指示灯
```

### 6.2 通信协议
| 接口 | 方法 | 请求 | 响应 |
|------|------|------|------|
| `/api/sensor` | GET | — | `{"temperature": 26.5, "humidity": 55.0}` |
| `/api/alert` | POST | `{"type": "on"}` 或 `{"type": "off"}` | `{"result": "ok"}` |

### 6.3 告警行为
- 收到 `alertOn` → 蜂鸣器响 10 秒后自动停止，LED 指示灯常亮
- 收到 `alertOff` → LED 指示灯熄灭
